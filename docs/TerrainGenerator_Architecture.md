# TerrainGenerator 概要设计

## 一、总体架构

系统采用**分层 + 模块化**架构，自顶向下分为五层：编排层、业务层、工具层、表现层、算法层。第三方库（GLFW / GLEW / OpenGL / ImGui / 字体）仅出现在最底层，不渗透到业务逻辑。

```
┌────────────────────────────────────────────────────────────────────┐
│  编排层（Orchestration）                                            │
│  main.cpp  ─── 主程序、窗口、主循环、纹理管理、事件分发              │
├────────────────────────────────────────────────────────────────────┤
│  业务层（Business）                                                 │
│  TerrainGenerator.{h,cpp}  ─── 地形生成核心、群系、统计、Seed       │
├────────────────────────────────────────────────────────────────────┤
│  工具层（Utility）                                                  │
│  Exporter.{h,cpp}  ─── PGM/PPM 导出、原生保存对话框                │
├────────────────────────────────────────────────────────────────────┤
│  表现层（Presentation）                                             │
│  UIStyle.{h,cpp}  ─── ImGui 主题、字体、控制面板、视口 overlay     │
├────────────────────────────────────────────────────────────────────┤
│  算法层（Algorithm）                                                │
│  PerlinNoise.{h,cpp}  ─── 2D 柏林噪声算法实现                       │
└────────────────────────────────────────────────────────────────────┘
       ▲
       │  第三方基础设施（仅在最底层，不渗透业务）
       │  GLFW · GLEW · OpenGL 3.0 · ImGui · JetBrainsMono TTF
```

## 二、模块依赖关系

```
                     ┌──────────────────────────────┐
                     │           main.cpp           │   ← 编排层
                     └──┬──────────┬────────────┬───┘
                        │          │            │
                        ▼          ▼            ▼
        ┌──────────────────┐  ┌────────────┐  ┌──────────────┐
        │ TerrainGenerator │  │  UIStyle   │  │   Exporter   │
        │   (业务层)       │  │ (表现层)   │  │  (工具层)    │
        └──┬───────────────┘  └─────┬──────┘  └──────────────┘
           │                       │
           ▼                       ▼
        ┌─────────────┐       ┌──────────────────────────────┐
        │ PerlinNoise │       │ ImGui · OpenGL · GLEW · GLFW │
        │  (算法层)   │       │       (第三方基础设施)        │
        └─────────────┘       └──────────────────────────────┘
```

**依赖原则**：

- **编排层 → 业务 / 工具 / 表现**：main.cpp 作为唯一的"组装者"，调用其他三个模块完成功能串联。
- **业务层 → 算法层**：业务层不直接依赖第三方图形库或 UI 库，依赖方向单向。
- **表现层 → 第三方基础设施**：表现层只通过 ImGui 接口工作，不直接调用业务模块。
- **工具层 → 无业务依赖**：Exporter 是纯 I/O 模块，可独立编译、独立测试。
- **算法层 → 零外部依赖**：PerlinNoise 仅依赖 C++ 标准库，便于独立验证与复用。

依赖箭头自上而下，**不存在反向依赖**，保证各层可独立替换与测试。

## 三、数据流

系统的主数据流为「**参数 → 生成 → 渲染 → 交互**」四段闭环：

```
  ┌─────────────┐    ① 参数设置     ┌─────────────────┐
  │  用户输入    │ ────────────────▶ │  TerrainParams  │
  │  (UI 控件)   │                   │  +  ViewState   │
  └─────────────┘                   └────────┬────────┘
                                               │ ② 触发重生成
                                               ▼
                                      ┌─────────────────┐
                                      │  业务层核心      │
                                      │  高度图 + 颜色图  │
                                      │  (heightMap /   │
                                      │   colorMap)     │
                                      └────────┬────────┘
                                               │ ③ 上传 GPU
                                               ▼
                                      ┌─────────────────┐
                                      │  OpenGL 纹理     │
                                      │  (纹理对象)      │
                                      └────────┬────────┘
                                               │ ④ 绘制到视口
                                               ▼
                                      ┌─────────────────┐
                                      │  右侧预览画面    │ ──┐
                                      └─────────────────┘   │ ⑤ 鼠标交互
                                                           │   (缩放/平移)
                                                           ▼
                                                    ┌─────────────┐
                                                    │  反馈到      │
                                                    │  ViewState   │
                                                    └─────────────┘
```

**关键数据通道**：

- **参数 → 业务层**：UI 控件修改参数对象，置位"需重生成"标志位。
- **业务层 → 表现层**：业务层产出 heightMap / colorMap 供表现层渲染。
- **业务层 → 工具层**：导出时由编排层调用工具层落盘。
- **表现层 → 编排层**：表现层仅置位请求标志（如"导出请求"），由编排层在帧末执行实际副作用。

## 四、模块职责

| 模块 | 层 | 职责 | 不做什么 |
|------|----|------|---------|
| `PerlinNoise.{h,cpp}`   | 算法层 | 实现 2D 柏林噪声的核心采样与多八度叠加；构造时根据种子初始化内部排列表 | 不感知地形、UI、文件、图形 |
| `TerrainGenerator.{h,cpp}` | 业务层 | 根据参数生成高度图与颜色图；维护群系配置表；提供统计缓存；提供 Seed 编解码、参数重置、预设等共享工具 | 不直接渲染、不读写文件、不操作 ImGui |
| `Exporter.{h,cpp}`      | 工具层 | 将 heightMap 写入 PGM；将 colorMap 写入 PPM；提供原生保存对话框 | 不感知业务参数含义，不依赖其他业务模块 |
| `UIStyle.{h,cpp}`       | 表现层 | 加载字体、配置主题、绘制控制面板各分区、绘制视口角落 overlay | 不调用业务模块的副作用，不持有业务状态 |
| `main.cpp`              | 编排层 | 创建窗口、初始化第三方库、组装各模块、运行主循环、管理 OpenGL 纹理、处理鼠标事件、缓存统计信息、执行导出副作用 | 不实现具体业务算法，不重定义 UI 主题 |

**职责边界的意义**：

- 业务层与表现层之间通过"标志位 + 引用参数对象"通信，避免双向耦合。
- 表现层"置位请求 + 编排层执行"的模式让 UI 保持纯渲染，副作用统一在主循环的可控时机发生。
- 算法层零外部依赖，未来可单独抽出做单元测试或移植到其他项目。

## 六、设计原则

### 1. 单一职责
每个模块只承担一类职责：算法层只算噪声，业务层只管地形数据，工具层只管文件，表现层只管 UI，编排层只做组装。模块名直接对应其层与职责。

### 2. 依赖单向
依赖方向自上而下，**禁止反向依赖**。业务层不知道有 UI 存在，算法层不知道有图形学存在。这一约束保证任何一层都可以独立替换或测试。

### 3. 业务与表现分离
- 业务逻辑（生成、统计、编解码）放在业务层；
- UI 控件只负责"显示"和"采集用户输入"；
- 副作用（文件导出、纹理刷新）由编排层在主循环的固定时机统一执行。

表现层通过"置位请求标志"而非"直接调用业务模块副作用"通信，使 UI 保持纯渲染，副作用可控。

### 4. 共享逻辑只维护一份
跨模块需要复用的纯函数（统计计算、配色公式、Seed 编解码、参数重置、预设表）统一放在业务层，避免散落在多个模块造成不一致。

### 5. 数据驱动的可配置项
群系阈值、颜色映射、Seed 位分配等可配置项都以数据表 / 静态常量形式集中声明，新增/调整只需改数据，无需改代码路径。

### 6. 渐进增强的交互体验
UI 风格（半透明、大圆角、动态缩放）、鼠标交互（滚轮缩放、拖动平移）、统计缓存等"锦上添花"的功能在不影响核心功能的前提下逐步叠加，便于排期与回滚。
